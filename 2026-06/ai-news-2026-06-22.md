## 论文与研究

### 1. Google DeepMind：AI Control Roadmap

- 日期：2026-06-18。
- 事件：Google DeepMind 发布 AI agent 安全路线，强调高能力 agent 需要通过 sandbox、endpoint security、prompt injection resistance、AI supervisors、实时阻断和审计来控制风险。
- 为什么重要：agent 安全正在从“模型对齐”扩展为“系统控制”，企业落地时必须同时考虑权限、隔离、监控和响应时间。
- 核验状态：已用 Google DeepMind 官方博客核验。
- 来源：[Google DeepMind - Securing the future of AI agents](https://deepmind.google/blog/securing-the-future-of-ai-agents/)

### 2. Agentic AI 架构综述

- 日期：arXiv 提交于 2026-01-18。
- 主题：论文提出 agentic AI 的架构与分类，把 agent 拆成 perception、brain、planning、action、tool use、collaboration 等模块，并讨论 MCP、native computer use、prompt injection 等问题。
- 为什么重要：这是理解 agent 系统设计空间的基础框架，有助于判断不同产品到底是在做模型、工具、执行环境还是协作协议。
- 来源：[arXiv:2601.12560](https://arxiv.org/abs/2601.12560)

### 3. AIDev：真实世界 AI Coding Agents 数据集

- 日期：arXiv 提交于 2026-02-09。
- 主题：论文聚合 932,791 个 agent-authored PR，覆盖 OpenAI Codex、Devin、GitHub Copilot、Cursor、Claude Code 等五类 agent，涉及 116,211 个 GitHub 仓库。
- 为什么重要：AI 编程 agent 已经进入真实软件工程流程，研究重点正在转向“实际 PR、review、issue、commit 数据”而不是只看离线 benchmark。
- 来源：[arXiv:2602.09185](https://arxiv.org/abs/2602.09185)

## 行业动态

### 1. 企业 agent 落地进入“运行环境竞争”

- 观察：Samsung 部署说明大型企业开始把 ChatGPT Enterprise / Codex 作为组织级平台；Ona 收购说明 OpenAI 在补企业 agent 的持久执行环境。
- 为什么重要：未来 agent 产品竞争会同时比拼模型、工具调用、云端持久运行、权限、审计、review 和数据边界。
- 来源：[OpenAI - Samsung deployment](https://openai.com/index/samsung-electronics-chatgpt-codex-deployment/)，[OpenAI - Ona](https://openai.com/index/openai-to-acquire-ona/)

### 2. Agent 安全成为基础设施问题

- 观察：DeepMind 的控制路线和 OpenAI 的企业执行环境指向同一结论：强 agent 不能只依赖模型承诺，还要靠系统边界、权限隔离、审计和自动化阻断。
- 为什么重要：这会影响企业采购、开发者框架、MCP 工具接入、浏览器/电脑控制和长期任务执行的设计。
- 来源：[Google DeepMind - AI agents security](https://deepmind.google/blog/securing-the-future-of-ai-agents/)
